# Kizuna (working title) — tenpo-ai

Voice-first AI Japanese learning app. iOS-first (SwiftUI), thin Node.js proxy backend.

The full product and technical spec lives in [docs/ARCHITECTURE.md](docs/ARCHITECTURE.md).
All major decisions are final and recorded in its Decision Log (§10).

## Repo layout

```
docs/ARCHITECTURE.md     The handoff spec this repo implements
ios/                     SwiftUI app (Xcode project generated by XcodeGen)
  project.yml            XcodeGen spec → run `xcodegen generate` in ios/
  App/                   App target: entry point, DI container, Home
  Packages/KizunaKit/    SPM package with all core modules (§4.2)
server/                  Node 22+ / Fastify API proxy (§7)
tools/                   Content import scripts (Phase 1)
COMPLIANCE.md            Privacy/licensing decisions record (§8)
```

## Getting started

### iOS app

```sh
cd ios
xcodegen generate          # regenerates Kizuna.xcodeproj from project.yml
open Kizuna.xcodeproj      # build & run the Kizuna scheme on an iOS 17+ simulator
```

Package tests run on macOS directly:

```sh
cd ios/Packages/KizunaKit && swift test
```

### Server (API proxy)

Requires Node ≥ 22 (uses native TypeScript type-stripping — no build step).

```sh
cd server
cp .env.example .env       # fill in secrets as providers are wired up
npm install
npm run dev                # http://localhost:8787/healthz
npm test
```

Provider API keys live only in `server/.env` (locally) / Fly secrets (deployed).
**Keys never ship in the iOS client** — non-negotiable (§7).
